Лабораторна робота №1: сайт-візитка «QA Mentor»

1. Тема, мета, місце розташування ЛР №1.1

Тема WEB-застосунку

Тема: Сайт-візитка «QA Mentor».

Мета створення WEB-застосунку

Мета: Створити персональний сайт-візитку наставника з тестування ПЗ (QA Mentor), який містить інформацію про досвід, послуги менторства, програми навчання, відгуки студентів та контактну форму для запису на консультацію.

Місце розташування ЛР №1.1 та репозиторії

  • Репозиторій з кодом сайту-візитки: посилання на GitHub
  • Жива сторінка сайту-візитки: посилання на GitHub Pages / Netlify
  • Репозиторій звітного HTML-документа: посилання на GitHub
  • Жива сторінка звітного HTML-документа: посилання на GitHub Pages

2. Опис предметного середовища та бізнес-логіки

Предметне середовище — ринок ІТ-освіти та менторства в галузі забезпечення якості програмного забезпечення. Ключові сутності: відвідувач сайту, ментор, програма навчання, консультація, відгук.

Сайт-візитка «QA Mentor» дозволяє відвідувачам ознайомитись із досвідом ментора, переглянути перелік послуг (індивідуальні сесії, міні-курси, ревʼю резюме, підготовка до співбесіди), залишити заявку на консультацію через форму зворотного звʼязку.

Функціональні вимоги

  • Відображення інформації про ментора, його досвід та стек технологій.
  • Секція з описом послуг та короткою програмою навчання.
  • Блок із відгуками студентів.
  • Контактна форма для відправки заявки на менторство.
  • Посилання на соцмережі (LinkedIn, Telegram, GitHub тощо).

Нефункціональні вимоги

  • Адаптивний інтерфейс для мобільних пристроїв.
  • Єдина кольорова палітра згідно індивідуального завдання.
  • Кросбраузерна сумісність (Chrome, Firefox, Edge).
  • Швидке завантаження сторінки (мінімум зовнішніх ресурсів).

3. Структура головної сторінки сайту-візитки «QA Mentor»

Головна сторінка містить такі семантичні блоки:

  1. <header> — логотип/імʼя ментора, навігація по секціях.
  2. <section> hero-блок з фото ментора та коротким описом.
  3. <section> «Про мене» з досвідом та скілами.
  4. <section> таблиця програм навчання та пакетів менторства.
  5. <section> відгуки студентів (списки/картки).
  6. <section> форма зворотнього звʼязку.
  7. <footer> контакти та копірайт.

Для оформлення використовується файл стилів style.css, де налаштовано кольорову палітру, сітку, відступи та адаптивність.

5.3. Теорія щодо тегу <img>

Тег <img> має обовʼязкові атрибути src (шлях до файла) та alt (текстовий опис). Атрибути width та height задають розмір, а через CSS можна зробити зображення круглим (наприклад, border-radius: 50%;).

Рисунок 3 – Маркований та нумерований списки

6.2. Теорія щодо тегів списків

Маркований список створюється тегом <ul>, нумерований — <ol>. Кожен елемент — це <li>. Атрибут type задає стиль маркерів або нумерації, start — початковий номер.

7. HTML-код форми

7.1. HTML-код форми запису на консультацію

<form class="contact-form" action="#" method="post">
  <label for="name">Ім'я:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">E-mail:</label>
  <input type="email" id="email" name="email" required>

  <label for="goal">Мета менторства:</label>
  <textarea id="goal" name="goal" rows="3"></textarea>

  <button type="submit">Надіслати заявку</button>
</form>

7.2. Демонстраційна форма у звітному документі

7.3. Теорія щодо тегу <form>

Форма створюється тегом <form> з атрибутами action (адреса обробника) та method (get або post). Усередині розміщуються поля <input>, <textarea>, <select>, <button> тощо.

8. Головна сторінка сайту-візитки «QA Mentor»

При відкритті сайту користувач бачить hero-блок із круглим фото ментора, його імʼям, роллю (QA Mentor) та коротким описом. Нижче розташовані блоки «Про мене», «Програми менторства», «Відгуки» та «Запис на консультацію».

Кольорова схема сайту: світлий фон для основного контенту, акцентні червоні елементи для кнопок запису, темно-коричневі заголовки та зеленуваті акценти в підкресленнях і рамках.
